# generated-middleware Specification

## Purpose

Define the behavior and contents of the Express middleware code generated by PayGate.

## Requirements

### Requirement: Target Node.js Express

Generated middleware SHALL target Node.js/Express routes only for this POC.

#### Scenario: Express route integration

- GIVEN a developer generated code for `/v1/data`
- WHEN they copy the integration snippet into an Express server
- THEN it shows `app.get('/v1/data', paywall, handler)`

### Requirement: Use MPP Express integration

Generated middleware SHALL use `mppx/express`, not `mppx/server`, for Express route middleware.

#### Scenario: Middleware imports

- GIVEN generated middleware is returned
- WHEN a developer reads the import section
- THEN it imports `Mppx` from `mppx/express`
- AND it imports `stellar` from `@stellar/mpp/charge/server`
- AND it imports `USDC_SAC_TESTNET` from `@stellar/mpp`

### Requirement: Use Stellar testnet USDC

Generated middleware SHALL charge Stellar testnet USDC only.

#### Scenario: Testnet charge configuration

- GIVEN generated middleware is returned
- WHEN the developer reads the MPP method configuration
- THEN `currency` is `USDC_SAC_TESTNET`
- AND `network` is `stellar:testnet`

### Requirement: Require recipient and secret environment variables

Generated middleware SHALL fail fast if `STELLAR_RECIPIENT` or `MPP_SECRET_KEY` are not configured.

#### Scenario: Missing recipient

- GIVEN the developer starts their server without `STELLAR_RECIPIENT`
- WHEN `mpp-middleware.js` is loaded
- THEN the server throws a clear PayGate error explaining that `STELLAR_RECIPIENT` is missing

#### Scenario: Missing MPP secret

- GIVEN the developer starts their server without `MPP_SECRET_KEY`
- WHEN `mpp-middleware.js` is loaded
- THEN the server throws a clear PayGate error explaining that `MPP_SECRET_KEY` is missing

### Requirement: Include replay-protection store

Generated middleware SHALL include a store for MPP charge replay protection.

#### Scenario: Store is present

- GIVEN generated middleware is returned
- WHEN the developer reads the `stellar.charge` configuration
- THEN it includes `store: Store.memory()`

### Requirement: Warn about response method limitation

Generated middleware SHALL tell developers to use `res.json()` so the `Payment-Receipt` header is attached correctly.

#### Scenario: Setup note explains response method

- GIVEN generated middleware is returned
- WHEN the developer reads the comments
- THEN the code warns that handlers must use `res.json()`
- AND explains that `res.send()` and `res.end()` are not patched

## Known Limitations

- Express only.
- Stellar testnet only.
- USDC only.
- Charge intent only.
- Uses in-memory store for the POC; production deployments should use a durable store if supported and needed.
